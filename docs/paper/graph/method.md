# 방법

MicroLad의 방법은 2D 조각 이미지에서 패턴을 배운 뒤, 그 패턴이 이어지는 3D 후보를 만드는 절차다. 목표 수치가 있으면, 만든 3D 후보에 후속 조정 단계가 붙는다.

Figure 1은 이 흐름을 세 구역으로 나눈다.

```text
a-d: 2D 패턴을 배우는 구역
e-i: 2D 패턴을 3D로 연결하는 구역
j: 목표가 있을 때 3D 후보를 후속 조정하는 구역
```

![Figure 1](../md/2508.20138v4-assets/figure-01-page-04.png)

그림의 기호 뜻은 다음과 같다.

```text
압축 장치 E: 이미지를 작은 표현으로 줄인다
복원 장치 D: 작은 표현을 이미지나 3D 구조로 펼친다
작은 표현 z: 압축된 숫자 묶음
노이즈 제거 모델 p_theta 또는 epsilon_theta: 흐트러진 표현을 되돌린다
오차 L: 목표와 얼마나 다른지 잰다
```

여기서 작은 표현은 사람이 보는 그림이 아니다. 모델이 계산하기 쉽게 줄여 놓은 숫자 지도다.

이 문서에서는 `E`와 `D`를 묶어 VAE라고 부른다. 작은 표현 `z`에서 노이즈를 지우며 2D 패턴을 배우는 모델을 LDM이라고 부른다. [2](#ref-2) [3](#ref-3)

## 학습과 생성의 순서

학습은 2D 조각 이미지 안에서 끝나고, 3D 후보는 그다음 생성 단계에서 만들어진다. 이 구분을 놓치면 VAE와 LDM이 3D 구조를 직접 학습한다고 오해하기 쉽다.

먼저 VAE가 2D 조각 이미지를 작은 표현으로 압축하고 다시 복원하는 법을 배운다. 이때 보는 것은 3D 부피가 아니라 2D 조각 이미지다. [2](#ref-2)

```text
2D 조각 이미지
-> VAE의 압축 장치 E
-> 작은 표현 z
-> VAE의 복원 장치 D
-> 다시 2D 조각 이미지
```

그다음 LDM이 이 작은 표현 `z`에서 노이즈를 지우는 법을 배운다. 여기서도 모델이 배우는 대상은 2D 조각 이미지에서 나온 작은 표현이다. 학습된 LDM은 흐트러진 2D 작은 표현을 2D 재료 패턴답게 되돌리는 방향을 익힌다. [3](#ref-3)

```text
2D 조각 이미지에서 나온 작은 표현 z
-> 노이즈 추가
-> 흐트러진 작은 표현 z_t
-> LDM이 노이즈를 예측
-> 2D 패턴을 되살리는 방향 학습
```

3D 후보는 이 두 학습이 끝난 뒤에 나온다. L-MPDD는 학습된 2D LDM을 3D 후보의 세 방향 단면에 반복해서 적용한다. 그래서 3D 후보는 3D 정답 데이터를 배워서 바로 나오는 것이 아니라, 2D 조각 이미지에서 배운 패턴을 세 방향으로 맞추며 만들어지는 결과다. [1](#ref-1) [4](#ref-4)

```text
학습:
2D 조각 이미지 -> VAE
2D 조각 이미지의 작은 표현 -> LDM

생성:
학습된 2D LDM -> 세 방향 단면에 반복 적용 -> 3D 후보
```

여기서 3D 후보를 고친다는 말은 VAE가 3D 구조를 직접 고친다는 뜻이 아니다. VAE는 2D 단면과 작은 표현을 오가는 장치이고, LDM은 세 방향에서 잘라 본 작은 표현들이 2D 조각 이미지에서 배운 패턴처럼 보이도록 노이즈를 줄이는 역할을 한다. [1](#ref-1) [4](#ref-4)

```text
VAE:
2D 단면 <-> 작은 표현

LDM:
세 방향 단면의 작은 표현을 2D 패턴 쪽으로 되돌린다

L-MPDD:
x, y, z 방향 단면을 번갈아 보며 같은 3D 후보를 조금씩 맞춘다
```

L-MPDD는 2D 모델을 3D 후보에 쓰기 위한 연결 방식이다. 3D 후보를 여러 방향으로 슬라이스해서 보고, 각 슬라이스가 2D 조각 이미지에서 배운 패턴처럼 보이도록 현재 3D 후보를 반복해서 고친다. 자세한 설명은 [[docs/paper/graph/mpdd|MPDD]] 노드에서 따로 다룬다. [1](#ref-1) [4](#ref-4)

## a-d: 2D 패턴을 배우는 구역

- a. 2D 재료 이미지 조각을 모은다.

  그림의 `x_2D` 묶음이다. MicroLad는 먼저 2D 단면 이미지에서 출발한다. [1](#ref-1)
- b. 이미지를 작은 표현으로 압축한다.

  `E(x)`는 압축 단계다. 2D 이미지를 그대로 쓰지 않고, 모델이 다루기 쉬운 작은 숫자 묶음으로 바꾼다. 논문에서는 이 작은 표현을 latent representation이라고 부른다. [1](#ref-1) [2](#ref-2)
- c. 작은 표현에 노이즈를 넣는다.

  그림의 `z_0 -> z_T` 과정이다. 깨끗한 작은 표현을 일부러 점점 흐트러뜨린다. 그래야 다음 단계에서 모델이 “흐트러진 표현을 되돌리는 법”을 배운다. [1](#ref-1) [3](#ref-3)
- d. 2D 모델이 노이즈를 예측하도록 학습한다.

  그림의 `epsilon_theta(z_t, t)`가 노이즈를 지우는 2D 모델이다. 논문에서는 이 모델 구조를 U-Net이라고 부른다. 학습이 끝나면 이 모델은 흐트러진 작은 표현에서 2D 재료 패턴을 되살리는 방향을 안다. [1](#ref-1) [3](#ref-3)

## e-i: 2D 패턴을 3D로 연결하는 구역

- e. 서로 이어지지 않은 2D 표현을 3D 후보의 초안으로 쌓는다.

  그림의 `z_T` 더미다. 이 단계의 조각들은 아직 서로 맞물려 있지 않다. 3D 구조를 만들기 위한 출발 재료다. [1](#ref-1)
- f. 3D 후보를 세 방향으로 잘라 보며 단면 패턴을 맞춘다.

  그림의 분홍, 빨강, 파랑 단면 묶음이다. L-MPDD는 같은 3D 후보를 세 방향으로 슬라이스해서 보고, 각 슬라이스가 2D 조각 이미지에서 배운 패턴과 맞도록 현재 후보를 반복해서 고친다. 한 방향에서 안 보이는 앞뒤 끊김은 다른 방향 단면에 드러난다. 그래서 세 방향을 모두 쓴다. [1](#ref-1) [4](#ref-4)
- g. 연결된 3D 작은 표현을 얻는다.

  그림의 `z_0` 큐브다. 이제 따로 놓인 2D 표현 더미가 아니라, 서로 맞물린 3D 표현이다. 아직 최종 3D 이미지는 아니고, 압축된 상태의 3D 구조다. [1](#ref-1) [4](#ref-4)
- h. 단면으로 펼쳤다가 다시 압축하며 어긋남을 줄인다.

  그림의 `D-E` 반복이다. `D`는 작은 표현을 단면 이미지로 펼치고, `E`는 그 단면을 다시 작은 표현으로 압축한다. 이 왕복을 세 방향에서 반복해 3D 구조의 어긋남을 줄인다. [1](#ref-1) [4](#ref-4)
- i. 최종 3D 재료 구조로 되돌린다.

  그림의 `x_3D`다. 디코더 `D`가 작은 3D 표현을 사람이 볼 수 있는 3D 재료 구조로 바꾼다. [1](#ref-1)

## j: 목표가 있을 때 3D 후보를 후속 조정하는 구역

j는 3D 후보를 새로 만드는 단계가 아니라, 이미 만든 후보에 붙는 목표 조정 단계다. 목표 수치가 있을 때 이 단계를 켜고, 안에서는 같은 작업을 반복한다. [1](#ref-1) [5](#ref-5)

```text
3D 후보에서 단면을 고른다
-> 단면을 작은 표현으로 압축한다
-> 목표 수치와 얼마나 다른지 계산한다
-> 작은 표현을 조금 바꾼다
-> 바뀐 단면을 3D 후보에 다시 넣는다
```

- `M`은 구조 수치다.

  부피 비율이나 표면적처럼, 구조 모양에서 바로 계산되는 값이다. [5](#ref-5)
- `H`와 `P`는 성능 수치다.

  `H`는 물리 계산 함수이고, `P`는 그 계산으로 얻은 성능 값이다. 예를 들어 “이 구조에서 물질이 얼마나 잘 지나가는가”를 계산하면 확산도 같은 성능 값이 나온다. [5](#ref-5)
- `L_SDS`, `L_M`, `L_P`는 세 가지 오차다.

  오차는 “지금 후보가 원하는 방향에서 얼마나 벗어났는가”를 재는 숫자다. `L_SDS`는 단면이 2D 데이터에서 배운 패턴을 벗어나지 않게 잡아준다. `L_M`은 구조 수치를 목표에 맞춘다. `L_P`는 성능 수치를 목표에 맞춘다. [5](#ref-5)

## 한 줄 요약

```text
VAE + LDM: 2D 패턴을 배운다
L-MPDD: 2D 패턴을 3D 후보로 연결한다
SDS: 목표가 있을 때 3D 후보를 후속 조정한다
```

이 방법의 첫 검증 대상은 [[docs/paper/graph/reconstruction|재구성]]이다. 그 다음 검증 대상은 [[docs/paper/graph/control|역방향 제어]]다. 재구성 노드에서는 L-MPDD가 만든 3D 후보가 원래 2D 데이터의 패턴 수치와 맞는지를 보고, 역방향 제어 노드에서는 목표 수치가 3D 후보를 원하는 방향으로 움직이는지를 본다.
