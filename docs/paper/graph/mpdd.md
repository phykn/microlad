# MPDD

MPDD는 3D 후보를 여러 방향으로 슬라이스해서 보고, 각 슬라이스가 2D 조각 이미지처럼 보이도록 현재 3D 후보를 반복해서 고치는 방식이다. 이 논문은 그 작업을 작은 표현 `z`에서 하기 때문에 L-MPDD라고 부른다. [1](#ref-1) [2](#ref-2)

## 왜 필요한가

2D 모델은 단면 한 장의 패턴을 배운다. 빈 공간이 어떻게 생기는지, 고체 물질이 어떻게 뭉치는지, 경계가 어떻게 이어지는지를 2D 이미지 안에서 배운다.

문제는 3D 후보다. 2D 단면을 그냥 여러 장 쌓으면, 각 장은 그럴듯해도 앞뒤 단면이 같은 부피 안에서 자연스럽게 이어진다는 보장이 없다.

```text
앞 단면: 같은 위치에 검은 영역이 있음
가운데 단면: 그 위치가 비어 있음
뒤 단면: 같은 위치에 검은 영역이 있음
```

한 장씩 보면 이상하지 않을 수 있다. 하지만 3D로 보면 같은 위치의 물질이 앞뒤로 이어지지 않는다. 앞뒤 연결이 어긋난 것이다.

MPDD는 이 문제를 한 방향에서만 보지 않는다. 같은 3D 후보를 앞-뒤, 좌-우, 위-아래 방향으로 계속 슬라이스해서 본다. [2](#ref-2)

## 슬라이스를 고친다는 뜻

슬라이스를 고친다는 것은 새 그림을 따로 만드는 것이 아니다. 현재 3D 후보 안에 있는 칸들의 값을 조금 바꾸는 것이다.

```text
1. 3D 후보를 만든다
2. 한 방향으로 슬라이스한다
3. 그 슬라이스를 2D 모델에게 보여준다
4. 2D 모델이 배운 패턴에서 벗어난 부분을 조금 줄인다
5. 바뀐 값이 원래 3D 후보에 반영된다
6. 다른 방향으로 다시 슬라이스한다
```

옆 방향 슬라이스는 앞쪽, 가운데, 뒤쪽을 한 번에 가로질러 본다. 그래서 앞 단면과 뒤 단면 사이가 끊겨 있으면, 옆 방향 슬라이스에서 그 끊김이 드러난다.

```text
앞 -> 가운데 -> 뒤
검정 -> 없음 -> 검정
```

이런 슬라이스는 자연스러운 2D 재료 이미지에서 배운 패턴과 잘 맞지 않는다. 2D 모델의 오차는 이 슬라이스를 더 자연스러운 쪽으로 바꾸도록 작동한다. 그 수정은 같은 3D 후보에 저장된다. 그래서 다음에 앞 방향으로 다시 잘라 보면, 가운데 단면도 바뀌어 있다.

핵심은 앞 단면과 뒤 단면을 직접 이어 붙이는 것이 아니다.

```text
같은 3D 후보를 공유한다
여러 방향 슬라이스가 그 후보를 가로질러 본다
각 방향에서 어긋난 부분을 조금씩 줄인다
그 수정이 같은 3D 후보에 누적된다
```

## SliceGAN과 다른 점

SliceGAN도 3D를 슬라이스해서 본다. 그래서 처음 보면 MPDD와 비슷해 보인다. 차이는 슬라이스 판단을 어디에 쓰는가다.

| 구분 | SliceGAN | MPDD / L-MPDD |
| --- | --- | --- |
| 슬라이스 사용 | 학습 때 평가 | 생성 중 반복 수정 |
| 판단자 | 2D 감별기 | 2D diffusion 모델 |
| 바뀌는 것 | 3D 생성기 | 현재 3D 후보 |
| 출력 방식 | 생성기가 3D 출력 | 후보를 계속 고침 |

SliceGAN은 3D 생성기를 훈련한다. 논문 부록의 SliceGAN 구조도 `Generator (3D)`와 `Discriminator (2D)`로 나뉜다. [4](#ref-4)

MPDD는 다르다. 이미 배운 2D diffusion 모델을 가지고, 지금 만들고 있는 3D 후보의 슬라이스를 계속 고친다. 판단 결과가 새 3D 생성기 훈련으로 가지 않고, 현재 3D 후보 자체에 바로 반영된다. [1](#ref-1) [2](#ref-2)

## 이 논문에서의 위치

이 논문에는 SliceGAN 같은 3D 전용 생성기가 따로 없다. 대신 2D 조각 이미지로 VAE와 2D LDM을 학습하고, 그 2D 모델을 L-MPDD로 3D 후보에 반복 적용한다. 마지막에는 복원 장치 `D`가 작은 표현을 3D 재료 구조로 펼친다. [1](#ref-1) [2](#ref-2)

넓게 보면 `D`는 구조를 펼치는 생성 역할을 한다. 하지만 SliceGAN처럼 3D 부피를 바로 뽑는 전용 generator를 따로 훈련하는 방식은 아니다. 이 논문의 3D 생성은 “3D 생성기 하나가 바로 출력한다”가 아니라, “2D 모델로 슬라이스를 계속 검사하고 현재 후보를 고친다”는 방식이다.

L-MPDD는 기존 MPDD보다 계산 위치도 다르다. 기존 MPDD는 픽셀 이미지에서 작업하지만, 이 논문은 작은 표현 `z`에서 작업한다. 논문은 이 차이로 64 x 64 x 64 부피 생성 시간이 기존 MPDD의 30분 이상에서 L-MPDD의 10초 미만으로 줄었다고 쓴다. [3](#ref-3)

따라서 방법 노드에서 L-MPDD가 맡는 일은 분명하다. 2D 조각 이미지에서 배운 패턴을, 여러 방향으로 잘라도 말이 되는 3D 후보로 옮기는 것이다.
