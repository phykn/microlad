# 논문의 주장

MicroLad는 2D 조각 이미지에서 3D 후보를 만들고, 그 후보를 목표 수치에 맞게 조정한다고 주장한다. [1](#ref-1) [2](#ref-2)

이 주장은 필요한 데이터 종류를 줄인다. 3D 정답 데이터가 많이 없어도 시작하고, “이 구조는 이 성능을 낸다”처럼 구조와 성능을 짝지은 데이터 없이도 목표 수치 쪽으로 후보를 조정한다. [2](#ref-2)

## 논문이 말하는 것

- 입력은 정답 표시가 없는 2D 조각 이미지다.

  논문은 하나의 2D 현미경 이미지에서 잘라낸 조각 이미지로 시작한다. 여기에는 “이 2D 조각이 실제 3D에서 어디에 있었는가”라는 정답 표시가 없다. [2](#ref-2)
- 출력은 3D 후보다.

  논문은 2D 이미지에서 배운 패턴으로 3D 부피를 만든다. 만든 3D 부피는 원래 2D 데이터의 패턴 수치와 맞아야 한다. 겉모양만 비슷하면 충분하지 않다. [1](#ref-1)
- 목표 수치는 부피 비율, 표면적, 확산도다.

  부피 비율은 특정 물질이 얼마나 많은지, 표면적은 서로 닿는 면이 얼마나 넓은지, 확산도는 물질이 얼마나 잘 지나가는지를 뜻한다. MicroLad는 이런 목표 수치를 넣고 3D 후보를 목표에 가까워지게 조정한다. [1](#ref-1) [3](#ref-3)

## 그림으로 보는 주장

![Figure 2](../md/2508.20138v4-assets/figure-02-page-05.png)

Figure 2는 두 흐름을 비교한다. 기존 흐름은 구조를 얻고 성능을 계산한다. MicroLad 흐름은 2D 데이터에서 3D 후보를 만들고, 목표 수치로 그 후보를 조정한다. [4](#ref-4)

```text
입력: 정답 표시가 없는 2D 조각 이미지
중간 결과: 패턴 수치가 맞는 3D 후보
목표: 부피 비율, 표면적, 확산도 같은 목표 수치
출력: 목표 수치 쪽으로 조정된 3D 후보
```

## 헷갈리기 쉬운 점

여기서 말하는 3D 구조는 실제 재료 안에 숨어 있던 하나뿐인 정답이 아니다. 2D 데이터와 패턴 수치가 맞는 3D 후보다. MicroLad의 주장은 그 후보를 만들고 목표 수치에 맞게 조정한다는 것이다. [1](#ref-1) [5](#ref-5)

이 주장을 실행하는 절차가 [[docs/paper/graph/method|방법]]이다. 핵심 단계는 2D 패턴 학습, 3D 연결, 목표 수치 조정이다.
